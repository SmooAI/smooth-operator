# smooth-operator docs — an Obsidian vault

This `docs/` folder is an **[Obsidian](https://obsidian.md) vault**. Open the
`docs/` directory as a vault and start at **[[Home]]** ([`Home.md`](Home.md)) —
the Map of Content that links every page. All pages also render on GitHub.

**New here?** [`Home.md`](Home.md) → `Concepts/`[`Overview.md`](Concepts/Overview.md)
→ `Guides/`[`Getting Started.md`](<Guides/Getting Started.md>).

## Layout

| Folder | What's in it |
| --- | --- |
| `Concepts/` | The mental model — what it is and why it's shaped this way. |
| `Architecture/` | The deep design docs (storage, ingestion, indexing, deploy, …). |
| `Guides/` | Task-oriented how-tos (getting started, self-hosting, integration, …). |
| `Reference/` | Lookup material (protocol, admin API, connectors, tools, config). |
| `Operations/` | Running it — access control, evals, observability, RBAC. |
| `Planning/` | Roadmap + feature-gap analysis. |

> The all-caps files at the root of `docs/` (e.g. `STORAGE.md`) are **redirect
> stubs** that point into the vault, kept so existing inbound links (from package
> READMEs and source doc-comments) don't break. The real content lives in the
> folders above.
